## 一、产品概述

### 1.1 一句话定义

巧记成长是面向家庭的亲子成长陪伴工具。家长和孩子一起设定目标，拆解为每天的卡片任务，孩子和TA的怪兽伙伴一起完成任务获得经验、见证伙伴成长——在陪伴中养成好习惯，在游戏中实现真目标。

### 1.2 核心原则

```
目标驱动一切。
宠物和游戏化是目标的放大器，不是替代品。
```

- 目标 → "我们要往什么方向走"
- 卡片 → "今天具体做什么"
- 积分 → "做到了得到什么"
- 伙伴 → "有伙伴陪我一起成长"

### 1.3 三级引擎模型

| 层级 | 引擎 | 职责 | 产出 |
|------|------|------|------|
| 顶层 | 🎯 目标引擎 | 设定方向、追踪进度、里程碑庆祝 | 目标卡、进度条、达成奖励 |
| 执行层 | 🃏 卡片引擎 | 每日任务分配、提交、验收 | 卡片实例、验收记录 |
| 反馈层 | ⭐ 激励引擎 | XP/积分/伙伴/成就四维反馈 | 伙伴状态、积分到账、等级变化 |

## 二、用户角色

### 2.1 角色定义

| 角色 | 描述 | 核心诉求 | MVP登录方式 |
|------|------|---------|------------|
| **家长**（Parent） | 家庭主账号，通常是父母 | 帮孩子养成好习惯、看到成长、少唠叨 | 手机号/邮箱注册 |
| **孩子**（Child） | 7-10岁小学生 | 有伙伴陪着、有奖励、能看到自己变厉害 | 同设备角色切换 |

### 2.2 家庭账户模型

```
Family（家庭）
  ├── Parent 账号（手机号/邮箱注册）
  ├── Child A（昵称 + 头像）
  └── Child B（昵称 + 头像）
```

**MVP规则：**
- 1个家庭 = 1个家长 + 至多3个孩子
- 家长创建孩子时只需填：昵称 + 选头像
- 孩子不建独立账号，通过家长端「切换至孩子模式」使用
- JWT携带 role(Parent/Child) 和 family_id

## 三、功能模块详述

### 模块A：目标系统（顶层引擎）

#### A-1 目标卡

| 字段 | 说明 | MVP |
|------|------|-----|
| 标题 | 目标名称，如"掌握分数加减法" | ✅ |
| 描述 | 具体说明 | ✅ |
| 周期 | 月目标 / 周目标 | ✅（仅月目标） |
| 状态 | 进行中 / 已达成 / 已过期 | ✅ |
| 关联卡片 | 关联的知识卡类型+成长卡 | ✅ |
| 进度 | 0-100%，按关联卡片完成度自动计算 | ✅ |
| 达成奖励 | 积分+XP+伙伴庆典 | ✅ |

**设定方式（三种）：**
1. 家长发起 → 孩子确认 → 激活
2. 孩子发起 → 家长审核 → 激活（Phase 1.5）
3. 共建 → 一起讨论确定（MVP推荐）

**正常流程：**
1. 家长进入「目标」页 → 点击新建目标
2. 填写标题、描述、周期（月/周）
3. 选择关联的卡片类型（如归因卡×每周2次、消化卡×每周1次）
4. 设定达成奖励积分额度（默认50分）
5. 发布 → 孩子确认 → 激活目标
6. 每天孩子完成关联卡片 → 进度自动推进
7. 进度25%/50%/75% → 伙伴弹出鼓励
8. 进度100% → 目标达成 → 发放奖励 + 伙伴全屏庆祝

**异常流程：**
| 场景 | 处理 |
|------|------|
| 孩子连续3天未完成关联卡片 | 伙伴温和提醒 + 家长端收到进度通知 |
| 目标周期到但未达成 | 标记「未达成」，可重新设定或调整周期 |
| 提前达成 | 正常发放奖励，可立即设定新目标 |
| 孩子想放弃 | 家长复盘 → 调整或替换目标，不惩罚 |

#### A-2 目标与卡片关联规则

- 一个目标可关联多张知识卡类型和成长卡
- 关联只设定「卡类型」和「频率」，不指定具体卡片实例
- 每日系统自动分配卡片时，优先分配有关联的卡片类型
- 进度计算：`已完成关联卡片数 / 计划总数 × 100%`
- 目标最多关联5张卡片（MVP限制，防止拆太散）

### 模块B：卡片系统（执行层）

#### B-1 卡片类型

| 系列 | 卡片名称 | 频率 | 创建者 | 提交方式 | 基础分 |
|------|---------|------|--------|---------|-------|
| 📚 知识卡 | 闪念卡 | 每日 | 系统模板 | 线上填写 | 2 |
| | 每日三件事 | 每日 | 系统模板 | 线上填写 | 2 |
| | 归因卡 | 随错随记 | 系统模板 | 线上填写 | 3 |
| | 消化卡 | 每2-3天 | 系统模板 | 线上填写 | 5 |
| | 费曼卡 | 每周 | 系统模板 | 线上填写 | 8 |
| 🌱 成长卡 | 习惯卡 | 每日重复 | 家长自定义 | 家长验收 | 2-3 |
| | 任务卡 | 单次/周期 | 家长自定义 | 家长验收 | 3-5 |
| 🎯 目标 | 目标卡 | 月/季度 | 家长+孩子共建 | 自动追踪 | — |

#### B-2 知识卡每日分配逻辑

```
每日分配（至多3张）：
  必选：每日三件事 + 闪念卡
  轮选（1张）：归因卡 / 消化卡 / 费曼卡（按周期轮换）
```

- 分配时优先匹配活跃目标关联的卡片类型
- 若某类型已关联目标，在卡片上显示所属目标名称+进度条

#### B-3 成长卡发布流程（家长端）

**正常流程：**
1. 家长进入「发卡片」页
2. 选择卡片类型（习惯卡 / 任务卡）
3. 填写标题、描述
4. 选择难度 ⭐/⭐⭐/⭐⭐⭐
5. 选择是否关联某个进行中的目标
6. 选择频率（每天 / 每周X次 / 单次）
7. 选择伙伴反馈效果（开心跳舞/冒星星/加油打气/爱心发射）
8. 指定孩子（多孩子时）
9. 发布

#### B-4 任务执行流程（孩子端）

**正常流程：**
1. 孩子打开「任务」Tab
2. 看到今日卡片清单（按优先级排列）
3. 伙伴在顶部显示今日状态
4. 点击卡片 → 查看详情
5. 知识卡：线上填写表单内容 → 提交
6. 成长卡：点击「我已做完」
7. 提交后 → 伙伴弹出鼓励动画 → "等爸爸妈妈验收！"
8. 伙伴头上出现「等待验收」标识

### 模块C：验收系统

#### C-1 验收流程

**正常流程（家长端）：**
1. 家长收到验收通知
2. 进入「工作台」→ 待验收列表
3. 选择一张卡片 → 查看孩子提交内容
4. ✅ 通过 → 可写点评 → 确认
5. 🔄 需改进 → 填写原因 → 退回

**通过后的反馈链：**
```
家长点「通过」
  → 伙伴跳庆祝动画（孩子端实时推送）
  → XP到账动画（+10 XP）
  → 积分到账动画（+基础分×难度系数×连续加成）
  → 目标进度更新（如有关联目标）
  → 连续打卡天数更新
```

**退回的反馈链：**
```
家长点「需改进」
  → 伙伴说「再试一次吧，我陪你」
  → 孩子端卡片状态变为「待改进」
  → 不扣分、不中断连续打卡
```
